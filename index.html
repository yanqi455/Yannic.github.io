Posted on 2020-06-22 Edited on 2020-07-14

##爬虫基础
####1.URL和URI区别
URI:Uniform Resource Identifier,即统一资源标志符
URL:Uniform Resource Locator, 统一资源定位符

URI=URL+URN

但是在目前的互联网中，URN用的非常少，所以几乎所有的URI都是URL
举个例子：
https://github.com/favicon.ico 是 GitHub 的网站图标链接，它是一个URL，也是一个URI。有这样一个图标资源，我们用URL/URI来指定了它的访问方式，
访问协议：https,访问路径（/即根目录）和资源名称：favicon.ico.

####2.超文本
网页源码都是超文本

####3.HTTP和HTTPS区别

HTTP：超文本传输协议（Hyper Text Transfer Protocol),Http是用于从网络传输超文本数据到本地浏览器的传送协议，它能保证高效而准确地传送超文本文档。

HTTPS：Hyper Text Transfer Protocol over Secure Socket Layer
即HTTP下加入SSL层，简称为HTTPs

####4.Http请求过程
1）浏览器向网站的服务器发送请求
2）网站服务器收到请求，进行解析和处理
3）返回响应，传给浏览器，响应包括页面源代码
4）浏览器再进行解析

网络与爬虫知识

Posted on 2020-06-14 Edited on 2020-09-26

1.了解一些网络的相关知识

1）url

http://kaoshi.edu.sina.com.cn/college/scorelist?tab=batch&wl=1&local=2&batch=&syear=2013
url：协议（http://) + 域名(kaoshi.edu.sina.com.cn) + 路由(college/scorelist,二级分类) + 参数（？以后为参数）

一般来说一个URL=协议+域名|IP+端口号+路径+参数+锚点
端口号：接在冒号后面，80端口提供http服务，443端口提供https服务，21端口提供FTP服务。
&： 表示分号，表示并列关系

2）线程，进程，协程，并行，并发

进程：正在运行的程序，比如你的系统正在运行着微信就是一个进程（一旦你的微信运行起来，系统就会分配内存和资源。

单进程：电脑只能做一件事，比如：电脑运行了微信以后就不能再打开淘宝。

多进程：就是可以同时运行多个程序。

其实一个单核cpu在一个时间点只能运行一个程序，之所以让你感觉你的电脑同时运行了多个程序，CPU在切换着做，并且切换速度之快让你觉着你所开的程序好像是在同时运行。

线程：在一个进程里可以执行多个任务，在这里的每一个任务就是线程。线程可以说是程序用CPU的一个基本单元，因此一个程序里面只是单一的一个执行路径，那么它就是单线程的，反之多个执行路径，就是多线程。

并行：一个单核的CPU，它每次只能执行一个进程，但是如果多核的CPU，同时可以执行多个进程，这就是并行。在同一个时间段可以执行多个进程。

并发：就是在一个时间点同时执行多个进程。例如：在双十一，多个用户同时描述一个商品就叫做并发。

互斥锁：在系统里面进程和进程之间互相独立的，而进程之间是互相独立的,而一个进程里面有多个线程。这时候线程就要搞事情了，毕竟一块蛋糕被共享了，谁都想多分一点，多抢占些资源，但同时就会导致数据不安全，所以这时候就有了互斥锁。

GIL：是python中控制线程执行权限的，所以当一个线程需要执行的时候，先要获得GIL之后才能执行，这样就不会产生安全问题。
在I/O流阻塞的时候，GIL会被释放。

我们的 Python 程序，如果是要处理多任务，想要充分的去利用我们多核的CPU，就开启多进程（multiprocessing）而对于一些输入输出的多任务，就开启多线程（threading.Thread、multiprocessing.dummy）？？？不太理解

协程（微线程）：在一个线程里面可以执行多个函数，线程和进程是通过系统调度的，而微线程则不需要可以通过需求自己调度，因为微线程是函数之间的切换，所以开销比较小。

3) python 线程池

传统多线程方案会使用“即时创建，即时销毁“的策略。尽管与创建进程相比，创建线程已经大大缩短时间，但是如果提交给线程的是执行时间短的任务，而且执行频率高。那么服务器将处于不停的创建线程，销毁线程的状态。

线程运行可以分为三个部分：1）线程的启动时间 2）线程的运行时间 3）线程的销毁时间
在多线程中，如果不能重复使用线程，那么意味着将会每次都要运行这三个步骤。势必会大大降低效率。
可以使用 ThreadPoolExecutor/queue 来实现线程池.

2.Chrome浏览器开发者工具

显示网页源代码/检查
显示网页源代码：静态源代码
检查：经过渲染和加载后显示的

显示网页源代码，检查
1.Elements : 页面渲染之后的结构，任意调整，即时显示；
2.console ：打印调试；
3.Sources ：使用的文件；
4.Network ：全部网络请求；
 XHR （XMLHttpRequest)： 异步请求，就是请求的一些数据,通过Http在浏览器和web服务器之间收发XML或其它数据，好处：可以动态更新网页，无需重新重新从服务器读取整个网页，也不需要额外的安装插件。

Get: 参数放到URL里的
Post：参数放到数据包里的， 一定要提交数据包
比如 注册，登录，这时候我们就是做post请求 post 的参数不会直接放到url上，会以form 表单的形式将数据提交给服务器。除XML之外，XMLHttp还能用于获取其它格式的数据，如JSON或者纯文本。

1.urllib

request: 用来发请求的
error: 在用request 模块遇到错了就可以用它来进行异常处理
parse: 用来解析URL地址的，比如解析域名地址，URL指定的目录
robotparse:这个用的比较少，用来解析网站的robot.txt

urlopen 默认是get请求，当data传递数据时，就为post请求
第一个参数url请求的链接
第二个参数data 就是专门给我们post请求携带参数的，比如登录时需要用户名和密码
第三个参数 timeout 就是设置请求超时时间如果等好久服务器都没有给我们返回数据，放弃请求。

request可以自定义请求方式（通过method方法)，此方法可以欺骗服务器自己是浏览器或者手机请求
